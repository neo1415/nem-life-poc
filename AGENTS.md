# NEM Life+ POC Agent Instructions

This project is the NEM Life+ proof of concept. The repository is the memory and the build-control files are the continuity system.

Before starting any task, read and obey docs/PRD.md, the current module PRD, and all relevant docs/steering files.

For any UI, frontend behaviour, report, PDF, customer screen, assessment screen, dashboard, or administrative interface task, also read and obey `docs/steering/14-ui-implementation-rules.md`, `docs/ui/*`, and all available UI reference files under `docs/ui-references/` or `docs/stitch_nem_life_visual_redesign/`.

After completing the task, re-read those documents and produce a compliance report.

If a module PRD conflicts with the steering documents, stop and report the conflict.

If a security requirement conflicts with a feature request, security wins unless explicitly approved after review.

Do not claim PASS if any required check was skipped.

Security wins over speed. Steering documents override convenience. No feature is done until it satisfies the PRD, steering, tests, verification, and reporting requirements.
