# Accessibility

The POC must meet practical WCAG 2.2 AA expectations where applicable.

Required: keyboard navigation, visible focus states, proper labels, semantic HTML, heading hierarchy, sufficient color contrast, no color-only meaning, accessible form errors, accessible dialogs/drawers, screen-reader-friendly progress, reduced motion support, responsive layouts, readable font sizes, and large mobile touch targets.

Every question screen must be usable without a mouse. Every CTA must have clear accessible text. Score colors must also use labels such as Covered, Review, Gap Found, or Not Sure.
