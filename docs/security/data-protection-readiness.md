# Data Protection Readiness

POC readiness:

- Data minimization rules are implemented.
- Prohibited POC data is centrally listed and regression-tested.
- Sensitive contact details are masked in admin/export surfaces.
- Demo reset and validation-on-load reduce accidental leakage.

Not production-ready:

- No formal DPIA.
- No production privacy notice.
- No DSR workflow.
- No breach process.
- No encryption/key-management review.
- No production access controls.
- No vendor/data-processing agreements.
